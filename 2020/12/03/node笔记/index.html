黑马的Node.js学习

-Node.js
-前端学习
Publish Date:   2020-12-03

第一天学习：

  • Node.js是什么

    • Node.js既不是语言，也不是框架，而是一个平台

  • Node.js中的JavaScript

    • 没有BOM和DOM

    • ECMAScript当中基本的JS语言部分

    • 核心模块：

      • 文件操作fs

      • http服务构建http

      • 路径操作path

      • 操作系统信息os

      • ……

        注：模块完全是封闭的，作用域是模块，没有全局作用域

    • 在Node中为JS提供了一些服务器级别的API

      • 文件操作的能力
      • http服务的能力

  • 搭建一个很简单的服务器 

    let http = require('http');
// 使用http.createServer()创建一个Web服务器
var server = http.createServer()
    // 收到request请求后，触发回调函数
server.on('request', function(request, response) {
        console.log('收到服务器请求了,请求路径是：' + request.url);
        // response.write可以用来给客户端发送响应数据
        // 最后一定要end(),相当于把话说完才接收
        if (request.url === '/') {
            response.end('this is the index page');
        } else if (request.url === '/login') {
            response.end('this is the login page');
        } else {
            response.end('404 not found')
        }
    })
    // 绑定端口号，启动服务器
server.listen(3000, function() {
    console.log('服务器启动成功了，可以通过http://127.0.0.1:3000来进行访问');
});

  • IP地址和端口号

    IP地址用来定位计算机，端口号定位到具体的应用程序

  • 关于Path模块

    • path.basename
      • 获取一个路径的文件名（默认包含扩展名）
    • path.dirname
      • 获取一个路径中的目录部分
    • path.extname
      • 获取一个路径中的扩展名部分
    • path.parse
      • 把一个路径转换为对象
        • root根路径
        • dir目录
        • base包含后缀名的文件名
        • ext后缀名
        • name不包含后缀名的文件名
    • path.join
      • 当你需要进行路径拼接的时候，推荐使用这个方法
    • path.isAbsolute
      • 判断一个路径是否为绝对路径

  • Node中的非模块成员

    在每个模块中，除了require、exports 等模块相关API之外，还有两个特殊的成员

    • __dirname :可以 动态获取 当前文件模块所属目录的绝对路径
    • __filename : 可以 动态获取 当前文件的绝对路径

    在文件操作中，使用相对路径是不可靠的，因为在Node中文件的相对路径 ——> node命令控制台所处路径

    为了解决这个问题，很简单，只要将相对路径转为__dirname或者__filename即可

    • 先找到当前文件所处的目录 node_modules 目录里面
    • node_modules/art-template
    • node_modules/art-template/packae.json文件

    注：如果main属性里面没有东西，或者没有package.json，则node回去找目录下的index.js。

第三天学习

  • 文件操作路径

    • ./data/a.txt 相当于当前目录
    • data/a.txt 相当于当前目录
    • /data/a.txt 相当于文件模块所处磁盘根目录。绝对路径

  • Express 的一些简单的API

    接收到get请求:

    // 基本路由
app.get('/', function(req, res) {
    res.send('hello world');
})

    接收到post请求:

    // 基本路由
app.post('/', function(req, res) {
    res.send('get a post request');
})

    静态服务：

    // 公开public文件夹，当请求以 /public/ 开头时，去 ./public/ 目录中找相应的资源
app.use('/public/', express.static('./public/'));  
// 当省略第一个参数的时候，则可以省略/public/
app.use(express.static('./public/'));

  • 在express使用 art-template 模板引擎

    • 安装：

      npm install --save art-template
npm install --save  express-art-template  （依赖于art-template）

    • 配置:

      // 配置使用art-template引擎
// 第一个参数表示，当渲染以 .art 结尾的文件时，使用art-template 引擎
// express-art-template 是专门在Express中把art-template整合的
app.engine('art', require('express-art-template'));
// 或者是：
app.engine('html', require('express-art-template'));
// render()方法，配置模板引擎后使用
// res.render('html模板名' ， {模板数据})
// 模板要放在views文件夹中，约定俗成的规定
app.get('/', function(req, res) {
    // 默认会去views目录中找index.html
    res.render('index.html', {
        title: 'hello world'
    })
})

    • 在Express里获取post表单

      安装：

      npm install body-parser --save

      配置：

      var express = require('express')
// 引包
var bodyParser = require('body-parser')
var app = express()

// 配置body-parser
// 加入这个配置，则req请求对象上会多出一个body
app.use(bodyParser.urlencoded({ extended: false }))
// parse application/json
app.use(bodyParser.json())

app.use(function (req, res) {
  res.setHeader('Content-Type', 'text/plain')
  res.write('you posted:\n')
  res.end(JSON.stringify(req.body, null, 2))
})

第四天学习

  • **回调函数 **

    • 不成立的情况

      function add(x, y) {
    console.log(1);
    // setTimeout是异步的
    setTimeout(function () {
        console.log(2);
        var ret = x + y;
        return ret;
    }, 1000);
    console.log(3);
}

console.log(add(10, 20));  // 1  3  undefined  2

    • 注意：凡是需要得到一个函数内部异步操作的结果，这种情况必须通过回调函数获取异步操作结果

      例如 setTimeout/readFile/writeFile/ajax等（回调函数就相当于一个通知，告诉你我执行完了）

      function add(x, y, cal lback) {
    // setTimeout是异步的
    setTimeout(function () {
        var ret = x + y;
        callback(ret);
    }, 1000);
}

add(10, 20, (ret) => {
    console.log(ret)
})

    • 基于原生XMLHttpRequest封装的get方法

      function get(url, callback) {
    var oReq = new XMLHttpRequest();
    oReq.onload = function() {
        callback(oReq.responseText);
    }
    oReq.open('get', url, true);
    oReq.send();
}

get('data.json', function(data) {
    console.log(data);
})

  • 设计Scheme发布Model

    var mongoose = require('mongoose');
var Schema = mongoose.Schema;
// 1.连接数据库
mongoose.connect('mongodb://localhost/mine');
// 2.设计集合结构
// 约束的目的就是不要有脏数据
const userSchema = new Schema({
    username: {
        type: String,
        required: true  // 必须有
    },
    password: {
        type: String,
        required: true
    },
    email: {
        type: String
    }
});
// 3. 将文档结构发布为模型
// 第一个参数： 传入大写名词单数字符串用来表示数据库名称
//              mongoose会自动将大写名词的字符串生成小写复数的集合名称
// 第二个参数：架构Schema
// 返回：模型构造函数
var user = mongoose.model('Mine', userSchema);

// 4. 有了模型构造函数之后， 就可以使用这个构造函数对 Users 集合中的数据为所欲为了（增删改查）

第五天

  • 回调地狱

    ​ 为了解决以上编码方式的问题，所以ES6新增了一个API：Promise

    let fs = require('fs');
// 创建Promise容器
// 1.Promise容器一旦创建，就开始执行里面的代码
var p1 = new Promise((resolve, reject) => {
    fs.readFile('./a.txt', 'utf-8', (err, data) => {
        if (err) {
            // 失败了，承诺容器里的任务失败了
            // 把容器里的pending状态变为rejected
            reject(err);
        } else {
            // 承诺容器中的任务成功了
            // 把容器的pending状态变为成功resolved
            resolve(data);
        }
    })
})

var p2 = new Promise((resolve, reject) => {
    fs.readFile('./b.txt', 'utf-8', (err, data) => {
        if (err) {
            reject(err);
        } else {
            resolve(data);
        }
    })
})

var p3 = new Promise((resolve, reject) => {
        fs.readFile('./c.txt', 'utf-8', (err, data) => {
            if (err) {
                reject(err);
            } else {
                resolve(data);
            }
        })
    })
    // p1就是那个承诺，p1成功了然后做指定的操作
p1
    .then(function(data) {
        console.log(data);
        return p2;
    })
    .then(function(data) {
        console.log(data);
        return p3;
    })
    .then(function(data) {
        console.log(data);
    })
    • 封装Promise (封装好后就不用写重复代码了)
    const promiseReadFile = function (path) {
    return new Promise((resolve, reject) => {
        fs.readFile(path, 'utf-8', (err, data) => {
            if (err) {
                reject(err);
            } else {
                resolve(data);
            }
        })
    })
}
promiseReadFile('./a.txt') 
    .then((function(data) {
        console.log(data);
        return promiseReadFile('./b.txt') ;
    })
     .then((function(data) {
        console.log(data);
        return promiseReadFile('./c.txt') ;
    })
      .then((function(data) {
        console.log(data);
    })

    第六天

  • 在express中配置使用 express-session 插件

    • 安装npm install express-session

    • 配置

      // 该插件会为req请求对象添加一个成员：req.session 默认是一个对象
app.use(session({
secret: 'keyboard cat',  // 配置加密字符串，在原有加密基础之上和这个字符串拼起来去加密，增加安全性
resave: false,
saveUninitialized: true  // 无论是否使用session，都默认直接给你分配一把钥匙
}))

    提示：默认session数据是内存存储的，服务器一旦重启数据就会丢失 ，真正的生产环境会把Session做持久化处理。
