第一章 准备工作

1.1 全局安装vue-cli3.0 脚手架

1）卸载：已经安装了旧版本执行 npm uninstall vue-cli -g

  1. 安装：npm install -g @vue/cli

3）查看版本：vue -V ，出现版本号即可

注：要先安装node.js

1.2 GIT基础命令

  • 拷贝项目： git clone <仓库地址>
  • 创建分支：git branch <name>
  • 创建并进入分支：git checkout -b <name>
  • 切换分支：git chckout <name>
  • 查看状态： git status
  • 添加所有文件： git add .
  • 提交： git commit -m '当前提交的描述'
  • 拉取： git pull
  • 推送：git push
  • 查看分支：git branch --list
  • 查看分支(包含远程分支)：git branch -a

1.3 工作当中的项目分支管理

  • master：主分支，一般都不会在此分支上进行项目开发

  • dev：开发分支

    • 版本分支：建立在dev分支下面

      需求 - 项目名 - Vx.x.x - 日期 来命名

      • Bug分支：建立在当前版本分支下面

        bug - bug的ID - 日期

1.4 构建项目

1） 命令

vue create <项目名>

2）VUE2.X与VUE3.X在目录结构上的区别

  • 去除了static、config、build，新增了public
  • 自动依赖node_modules
  • 默认配置webpack，通过vue.config.js修改
  • vue inspect可以查看webpack的默认配置

3）项目大致结构

2.3 VUE文件规则

1）固定的三个内容：template、script、style

  • template ： 必须要有一层父级元素，否则会报错。元素不一定是Div，其他也可以
  • script：
    • name : 当前的名称
    • components： 组件，引入其他组件
    • data： 数据，v-model绑定数据使用
    • created、mounted生命周期
    • methods： 定义函数
    • props、watch：组件通信
    • ……

2）Vue基本语法 （参考文档https://cn.vuejs.org/v2/api/）

  • v-model： 在表单控件或者组件上创建双向绑定 只支持input、select、textarea、component
  • v-for
  • v-show与v-if区别： v-if 直接删除DOM元素，而v-show只修改display
  • v-bind:class 或 :class
  • 数据绑定的方式
    • :class = "{'active': isActive}" // 当isActive为true时就添加class属性，否则不添加
    • :class = "{'active': isActive, 'sort': isSort}" // 也可以放多个

2.5 添加输入框验证规则

1）简单的添加验证规则

let reg = /^([a-zA-Z]|[0-9])(\w|\-)+@[a-z0-9A-Z]+\.([a-zA-Z]{2,4})$/; // 邮箱的正则
let reg = /^(?!\D+$)(?![^a-zA-Z]+$)\S{6,20}$/                  // 密码的正则 6-20数字加字母
let reg = /^[a-z0-9]{6}$/                                      // 验证码的正则  6位数字或字母
// 使用时调用reg.test(字符串)即可

2）在src下新建utils文件夹，存放验证规则

以validate.js为例 （使用时直接 import { stripscript,xxx } from '@/utils/validate.js'即可）

把这里的验证作为一个公共的东西来用

/* 
* 过滤特殊字符
*/
// 暴露函数
export function stripscript(s) {   
  var pattern = new RegExp("[`~!@#$^&*()=|{}':;',\\[\\].<>/?~！@#￥……&*（）&;—|{}【】‘；：”“'。，、？]")
  var rs = "";
  for (var i = 0; i < s.length; i++) {
    rs = rs + s.substr(i, 1).replace(pattern, '');
  }
  return rs;
}

2.6 改为VUE3.0语法

1）准备工作

  • 安装依赖 npm install @vue/composition-api --save

  • 在main.js下添加 

    import VueCompositionApi from '@vue/composition-api'; 
Vue.use(VueCompositionApi);

  • 在组件中引入 compositionApi 

    import { reactive, ref, isRef, toRefs, onMounted, watch, onUnmounted } from '@vue/composition-api';

2）setup 函数

  • 按照官方给出的说法，setup函数是一个新的Vue组件选项，是用于在组件中使用Composition API的入口。
export default {

  setup(props, context) {
   // 这里放data数据、生命周期、自定义的函数
   context.attrs    // 等同于 this.$attrs 
   context.slots    
   context.parent
   context.root     // this 
   context.emit     // this.$emit
   context.refs     // this.refs
      ....
  }
}

3）reactive （声明一个单一对象）

  • 取得一个对象并返回原始对象的响应数据处理。
    • const obj = reactive({ count: 0 })

4）ref（声明基础数据类型）

  • 内部值并返回一个响应性且可变的ref对象。ref对象具有.value指向内部值的单个属性。
    • const number = ref(0);
    • 获取值方式：number.value

5）isRef 和 toRefs

  • isRef : 检查一个对象是否是ref对象：
    • const unwrapped = isRef(foo) ? foo.value : foo;
  • toRefs : 将reactive对象转换为普通对象，保证对象解构或拓展运算符不会丢失原有响应式对象的响应。
function useMousePosition() {
  const pos = reactive({
    x: 0,
    y: 0
  });
  return toRefs(pos);
}
const { x, y } = useMousePosition();

2.7 Axios拦截器和跨域

1）设置拦截器

  • 在utils下创建request.js （设置拦截器）
import axios from 'axios'
// 在生产模式的时候请求地址设置为 /devapi
const BASEURL = process.env.NODE_ENV === 'production' ? '' : '/devApi';
// 创建axios，赋给变量service
const service = axios.create({
    baseURL: BASEURL, // 拼出来是 http://192.168.31.102:8080/devApi
    timeout: 1000
});

// 添加请求拦截器
service.interceptors.request.use(function(config) {
    // 在发送请求之前做些什么
    return config;
}, function(error) {
    // 对请求错误做些什么
    return Promise.reject(error);
});

// 添加响应拦截器
service.interceptors.response.use(function(response) {
    // 对响应数据做点什么
    return response;
}, function(error) {
    // 对响应错误做点什么
    return Promise.reject(error);
});

export default service;
  • 拦截器的步骤如下所示：

拦截器的概念参考 https://www.jianshu.com/p/a0c67f4e145e

在request.js当中解决的是一个拦截器的问题，每一次你进行请求和响应的时候都会对其进行一次拦截和处理

在api.js当中进行的就是具体的请求操作

2）设置跨域

  • 设置代理

    proxy: {
  '/devApi': {
      target: "http://www.web-jshtml.cn/productapi", // 你请求的第三方接口
      changeOrigin: true, // 在本地会创建一个虚拟服务端
      pathRewrite: { // 路径重写，
        '^/devApi': '' // 替换target中的请求地址
      }
   }
}, // 设置代理

    在遇到/devapi的时候，将头部的本地IP 192.168.XXX.XXX设置为target，将devApi替换为空

4.4 VUEX

1）VueX状态图

  • state

    储存单一状态，存储的基本数据

  • getters

    对State 里面的数据二次处理（对数据进行过滤类似filter的作用）比如State返回的为一个对象，我们想取对象中一个键的值用这个方法

    取用方法：this.$store.getters.xxx

  • mutations

    对数据进行计算的方法全部写在里面（类似computed） 在页面中触发时使用
    this.$store.commit(‘mutationName’)触发Mutations方法改变state的值

  • actions

    actions功能和mutations是类似的，都是改变store里面的state，不过与mutations的两点不同：

    • action主要是异步的操作，mutation必须同步操作

  • modules

4.5 浏览器端存储

1） Cookie

  • 安装依赖 npm i cookie_js
  • 储存
    • cookie.set(‘key’, ‘value’);
    • cookie.set({key1: ‘value1’, key2: ‘value2’});
  • 获取
    • cookie.get(‘key’);
    • cookie.get([‘key1’, ‘key2’]);
  • 清除
    • cookie.remove(‘key’);
    • cookie.remove(‘key1’, ‘key2’);
    • cookie.remove([‘key1’, ‘key2’]);

2）HTML本地存储（sessionStorage） 小的交互

  • 存储
    • window.sessionStorage.setItem(“key”, “value”);
  • 获取
    • window.sessionStorage.getItem(“key”);
  • 删除
    • window.sessionStorage.removeItem(“key”);
  • 清空所有
    • window.sessionStorage.clear()

4.6 路由守卫

顾名思义，守卫路由，路由不可以想跳转就跳转，比如控制页面只有在登录后才可以跳转。

（参考https://router.vuejs.org/zh/guide/advanced/navigation-guards.html#%E5%85%A8%E5%B1%80%E5%89%8D%E7%BD%AE%E5%AE%88%E5%8D%AB）

1） 在router文件夹中添加defend.js

router.beforeEach((to, from, next) => {
  // beforeEach只有在路由改变的时候才触发
  // to 指的是下一个页面
  // from 指的是上一个页面

})

2）token令牌

  • 直接进入index的时候，参数的to被改变成了’/index’，触发路由指向，就会跑beforeEach

  • 查看token，如果没有token则判断白名单

    • 若目标路由不在白名单里面 ===> next指向login，再次发生路由指向，再跑beforeEach，

      to被改成了’/login’

    • 白名单判断存在，则直接执行next(),因为没有参数，所以不会再beforeEach

  • ========> 避免了死循环问题
