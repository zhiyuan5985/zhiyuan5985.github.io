Vue 原理

  • 组件化基础
  • 传统组件，只是静态渲染，更新还需要依赖 DOM 操作
  • 数据驱动视图 —— Vue MVVM （数据操作DOM）

MVVM （通过修改 Model 中的数据，去操作 视图的变化，而不用再去操作 DOM）

Vue 响应式

  • 组件 data 的数据一旦变化，立刻触发视图的更新

  • 实现数据驱动视图的第一步

  • 考察 Vue 原理的第一步

  • 核心 API （Object.defineProperty）

    function observer(target) {
    // 只监听对象或者数组
    if (typeof target !== 'object' || target == null)  return target
    // 重新定义各个属性（for in），进行监听
    for (let key in target) {
        defineReactive(target, key, target[key])
    }
}
function defineReactive(target, key, value) {
    // 给每个对象的键都绑定属性
    Object.defineProperty(target, key, {
        get() {
            return value
        },
        set(newValue) {
			if (newValue !== value) {
                value = newValue
                updateView()    // 进行视图更新，会有输出提示
            }
        }
    })
}

  • 复杂对象，深度监听

    // 定义复杂的数据
const data = {
    name: 'zhangsan',
    age : 20,
    info: {
        address: '北京'    // 需要深度监听
    }
}
    function observer(target) {
    // 只监听对象或者数组
    if (typeof target !== 'object' || target == null)  return target
    // 重新定义各个属性（for in），进行监听
    for (let key in target) {
        defineReactive(target, key, target[key])
    }
}
function defineReactive(target, key, value) {
    // 深度监听
    observer(value)
    // 给每个对象的键都绑定属性
    Object.defineProperty(target, key, {
        get() {
            return value
        },
        set(newValue) {
			if (newValue !== value) {
                // 深度监听, 防止赋值的时候变成复杂对象
                observer(value)
                value = newValue
                updateView()    // 进行视图更新，会有输出提示
            }
        }
    })
}

  • 几个缺点

    • 深度监听需要递归到底，如果对象数据过深的话，一次性计算量大
    • 无法监听新增 / 删除属性 （Vue.set 和 Vue）

虚拟 DOM

  • 虚拟 DOM 是实现 vue 和 react 的基石，diff 算法是 vdom 中最核心，最关键的部分
  • 用 JS 模拟 DOM 结构，计算出最小变更之后，再操作 DOM
  • 新旧 vnode 对比，得到最小的更新范围，最后更新 DOM

// h 函数 ，创建虚拟 DOM
var vnode = h('节点名称', '属性', '子节点数组')
// patch 函数，把虚拟 DOM 渲染在节点上
patch(container, vnode)
// patch 函数，更新已有的虚拟 DOM
patch(vnode, newVnode)

diff 算法

  • 只比较同一层级的元素，不跨层比较

  • tag 不相同，则直接删掉重建，不再深度比较

  • tag 和 key 两者相同，则认为是相同节点，不再深度比较

模板编译

  • vue的模板到底是什么，即模板怎么转为 js 代码？
    • 模板编译为 render 函数， 执行 render 函数返回 vnode
    • 基于 vnode 再执行 patch 和 diff
  • 也可以用 render 代替 template

初次渲染的过程

  • 解析模板为 render 函数（或在开发环境已完成， vue-loader）
  • 触发响应式，监听 data 属性 getter， setter
  • 执行 render 函数， 生成 vnode， patch(elem, vnode)

更新过程

  • 修改 data， 触发 setter （此前在 getter 中已被监听）
  • 重新执行 render 函数， 生成 newVnode
  • patch(vnode, newVnode)

异步渲染原理

  • 汇总 data 的修改，一次性更新视图
  • 减少 dom 的操作，提高性能
  • 如果想要立即获得修改后的 DOM 元素， 则要使用 $nextTick

前端路由原理

  • 回顾 vue-router 的路由模式
    • hash模式
    • H5 history 模式
  • hash 的特点
    • hash 变化会触发网页的跳转，即浏览器的前进后退
    • hash 变化不会刷新页面，SPA必须的特点
    • hash 永远不会提交到 server 端
    • 通过 window.onhashchange 来监听的
  • H5 History 的特点
    • 用 url 规范的路由， 但跳转时不刷新页面
    • history.pushState 进行路由跳转
    • history.onpopstate 浏览器的前进后退

面试真题

  • v-show 和 v-if 的区别

    • v-show 通过 css display 控制显示和隐藏
    • v-if 是组件真正的渲染和销毁，而不是显示和隐藏
    • 频繁切换显示状态用 v-show，否则用 v-if

  • 为何在 v-for 当中使用 key

    • 必须用 key， 且不能是 index 和 random
    • diff 算法中通过 tag 和 key 来判断，是否是 sameNode
    • 减少渲染次数，提升渲染性能
