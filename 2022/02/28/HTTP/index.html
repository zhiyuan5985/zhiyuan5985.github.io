HTTP
发布日期:   2022-02-28

HTTP常见的状态码有哪些

  • 1xx：服务器收到请求
  • 2xx：请求成功，如 200
    • 200： 成功
  • 3xx：重定向，如 302
    • 301：永久重定向
    • 302：临时重定向
    • 304：资源在本地还有效
  • 4xx：客户端错误，如 404
    • 403：没有权限
    • 404：资源未找到
  • 5xx：服务端错误，如 500
    • 500：服务器错误
    • 504：网关超时

HTTP Methods

  • 传统的 API 设计：把每个 url 当做一个功能
    • get 获取数据
    • post 新建数据
    • patch / put 更新数据
    • delete 删除数据
  • Restful API ：把 每个 url 当做一个唯一资源

HTTP Headers

  • Request Headers
    • Accept 浏览器可接收的数据格式
    • Accept-Encoding ：浏览器可接收的压缩算法，如 gzip
    • Accept-Language： 浏览器可接收的语言，如 zh-CN
    • connection： keep-alive 一次TCP连接重复使用
    • cookie
    • Host
    • User-Agent：浏览器信息
  • Response Headers
    • Content-type ： 返回的数据类型
    • Content-length ：返回数据的大小，多少字节
    • Content-Encoding：返回数据的压缩算法
    • set-Cookie：服务端改 cookie
  • 缓存相关的 Headers
    • Cache-Control
    • Last-Modified
    • Etag

HTTP 缓存

  • 哪些资源可以被缓存？

    • —— 静态资源 (js、css、img)

  • 强制缓存

  • cache-control 的值

    • max-age ： 设置缓存的过期时间
    • no-cache：不用强制缓存
    • no-store：不用缓存

  • Expires

    • 同在 response headers 里

    • 已被 cache-control 替代

  • 协商缓存

    • 服务器端缓存策略 （服务端来判断这个文件是否需要缓存）

    • 服务器判断客户端资源，是否和服务端资源一样

    • 一致则返回 304，否则返回 200 和最新的资源

  • 资源标识（两种）

    • Last-Modified 资源的最后修改时间， 只能精确到秒级
    • Etag 资源的唯一标志(一个唯一字符串)，优先使用 Etag

  • 三种刷新操作
    • 正常操作（访问链接）：强制缓存有效，协商缓存有效
    • 手动刷新（f5、cmd+r）：强制缓存失效， 协商缓存有效
    • 强制刷新（ctrl+f5 / cmd+shift+r）:强制缓存失效，协商缓存失效

HTTPS协议

  • HTTP 和 HTTPS 的区别

    • HTTP 是明文传输，敏感信息容易被中间劫持
    • HTTPS = HTTP + 加密，劫持了也无法解密
    • 现代浏览器开始强制 https 协议

  • 加密方式

    • 对称加密：一个 key 同时负责加密和解密
    • 非对称加密：一对key，A加密后，用B解密；B加密后，用A解密

  • https 证书

    • 浏览器校验证书，防止中间人攻击
